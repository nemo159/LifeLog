# 기능 명세서: 확장 기능 (Enhanced Features)

**문서 ID**: `specs/002-enhanced-features.md`  
**작성일**: 2026-01-10  
**상태**: Draft  
**참조**: `specs/overview.md` Phase 2~4

## 개요
MVP 이후 추가될 사용자 경험 개선 및 고급 기능을 정의합니다.

## 상세 요구사항 (Requirements)

### 1. 타임라인 및 조회 개선 (View Improvements)

**REQ-VIEW-001: 타임라인 그룹화 (Timeline Grouping)**
- **설명**: 타임라인의 기록들을 년(Year), 월(Month) 단위 헤더로 묶어서 표시해야 한다.
- **UI**: 스크롤 시 날짜 헤더가 고정되거나(Sticky Header), 명확히 구분되어야 한다.
- **관련**: 사용자 요청 7번.

**REQ-VIEW-002: 캘린더 뷰 (Calendar View)**
- **설명**: 월간 달력 형태로 기분 상태를 한눈에 볼 수 있어야 한다.
- **기능**:
  - 날짜별로 대표 기분 아이콘(또는 색상) 표시.
  - 날짜 클릭 시 해당 일자의 기록 목록으로 이동하거나 팝업 표시.
- **관련**: 사용자 요청 6번.

**REQ-VIEW-003: 정렬 필터 (Sort Filter)**
- **설명**: 타임라인을 '최신순', '오래된순'으로 정렬할 수 있어야 한다.
- **관련**: 사용자 요청 4번.

### 2. 에디터 확장 (Editor Enhancements)

**REQ-EDIT-001: 태그 및 이벤트 (Tags & Events)**
- **설명**: 기분 점수 외에 구체적인 활동이나 감정을 태그로 추가할 수 있어야 한다.
- **예시**: #운동, #독서, #친구만남, #우울함, #즐거움.
- **UI**: 칩(Chip) 형태로 입력 및 선택.
- **관련**: 사용자 요청 10번.

**REQ-EDIT-002: 위치 및 날씨 자동 태깅 (Auto Location & Weather)**
- **설명**: 기록 작성 시 현재 위치와 날씨 정보를 자동으로 가져와 입력해야 한다.
- **조건**:
  - 사용자 위치 권한 허용 시 동작.
  - OpenWeatherMap 등 무료 날씨 API 활용 고려.
  - 수동 수정 가능해야 함.
- **관련**: 사용자 요청 11번.

### 3. 시스템 및 데이터 (System & Data)

**REQ-SYS-001: 테마 변경 (Theming)**
- **설명**: 앱의 전반적인 색상 테마를 변경할 수 있어야 한다.
- **옵션**: 시스템 설정 따르기, 라이트 모드 고정, 다크 모드 고정, 포인트 컬러 선택(예: Blue, Green, Pink).
- **관련**: 사용자 요청 8번.

**REQ-SYS-002: 통계 (Statistics)**
- **설명**: 저장된 기분 데이터를 바탕으로 통계를 제공해야 한다.
- **내용**: 월별 기분 평균, 기분 분포(5점 며칠, 1점 며칠 등), 가장 많이 사용한 태그.
- **관련**: 사용자 요청 9번.

**REQ-SYS-003: 구글 드라이브 백업 (Google Drive Backup)**
- **설명**: 로컬 DB 및 이미지 파일을 사용자의 구글 드라이브 앱 폴더(App Data Folder)에 백업하고 복원할 수 있어야 한다.
- **방식**: 수동 백업 버튼 제공 또는 주기적 자동 백업(옵션).
- **관련**: 사용자 요청 5번.

## 우선순위 (Implementation Priority)

1. **Phase 1 (MVP 완료)**: 사진, 삭제, 상세, 정렬 (기존 진행 중 항목).
2. **Phase 2 (UX)**: 그룹화, 캘린더, 테마.
3. **Phase 3 (Content)**: 태그, 위치/날씨, 통계.
4. **Phase 4 (Cloud)**: 백업.
